# 要件とスコープ

## このドキュメントで分かること

- **このプロダクトで最初に達成したいこと（MVP）**：ブラウザで雨雲を見る、といった範囲の線引き。
- **やらないこと**：予測モデル自前開発や課金アプリなど、初期では対象外のもの。
- **守るべきルールの例**：本番でフェイクデータを使わない（MUST NOT）、デプロイ先の前提など。

## 目的

ユーザーが**ブラウザまたはモバイル Web**から、現在地または任意地点を中心に**近実時間の雨雲（降水強度）**を地図上で確認できるようにする。データは**公的または許諾済みのソース**に限定する。

## 初期スコープ（MVP）

- 1 地域（例: 日本国内）にフォーカスした表示
- **過去数時間〜現在に近いタイムスライス**の閲覧
- **タイムラインの短いアニメーション（任意だが MVP 内に条件付きで含める）**: 同時に保持・連続再生するフレームは **最大 24 本**、再生間隔は **1 フレームあたり 500ms〜2000ms**（実装でこの範囲に収める）。それを超える要件は MVP の後に回す。
- 地図ベースライン（OSM 等）の上に**降水オーバーレイ**
- 単純な認証なし・個人情報の収集は設計上も最小（アクセスログのみ等）

### デプロイ前提（規範）

MVP の **物理ホスティング**は **GitHub Pages（静的フロント）＋ Cloudflare Workers（API・キャッシュ・定期処理）** とする（`09-deployment-github-pages-and-cloudflare.md`）。  
**ひとことで**：画面は GitHub Pages の静的サイト、裏側の API と定期処理は Cloudflare Workers 上に置く、という前提で設計されています。

## 非目標（初期ではやらない）

- 独自の物理シミュレーションや予測モデルの開発
- マルチテナント向け課金・アカウント体系
- オフライン完結アプリ（ネイティブのストア配信）

**ひとことで**：「科学シミュレーション」「SaaS 課金」「アプリストア向けオフラインアプリ」は、最初のリリース対象に含めません。

## フェイクプロバイダと環境分離（設計上 MUST）

**規範（normative）**＝守るべきルール、**参考（informative）**＝読み物・例、という使い分けは索引（`00-index.md`）に従います。ここでは **MUST NOT**＝**してはならない**、**SHOULD**＝**した方がよい**、という意味で使います。

開発・CI・ステージングでは、**フェイク／フィクスチャプロバイダ**（本物そっくりのテスト用データ源）を用いてよい。

- **本番環境ではフェイクプロバイダを有効にしてはならない（MUST NOT）**。起動時または設定読込時に、環境名（例: `production`）とプロバイダ種別の組み合わせを検証し、違反時は**起動・デプロイを失敗**させるガードを設ける要件とする（実装詳細は実装フェーズ。Cloudflare Workers では `wrangler deploy` 前チェックや実行時拒否など）。
- ステージングでフェイクを使う場合は、**利用規約表示・属性表示**を本番と同一ロジックで試せるよう、UI 上でデータソースがフェイクであることを明示できることが望ましい（SHOULD）。

**ひとことで**：テスト用の偽データは開発では OK ですが、**本番では絶対にオンにしない**。設定ミスを防ぐチェックを入れるのが要件です。

## 成功基準（設計レベル）

- データソースを**差し替え可能**（テスト用スタブ含む）
- バックエンドとフロントの**契約（メタデータ JSON／タイル URL 規約）が安定**し、**契約バージョン**で後方互換を管理する
- **メタデータが示すフレーム一覧・coverage・ズーム範囲と、実際に取得可能なタイル**の間に、設計上許容される不整合がないこと（許容しない。部分ズームや部分範囲のみ先行公開する場合は `03` の公開モデルに従い、メタデータ側で「利用可能 coverage / zoom」を表現する）
- 更新間隔・遅延・キャッシュが文書化され、運用でチューニング可能

**ひとことで**：**入れ替え可能なデータ源**、**フロントと API の約束（JSON）がブレない**、**メタに書いたことと実際に取れるタイルが矛盾しない**、**運用で調整できる説明がある**、を満たすことを目指します。

以下は、各 rev で設計に反映したメモです（履歴としてそのまま残しています）。

## レビュー反映（rev.2）

- 「公的」のみに依存すると検証が難しいため、**開発用フェイクプロバイダ**を要件に明記（上記のとおり本番では MUST NOT）。
- スコープを「静止画のみ」と曖昧にしない。**タイムディメンション**は MVP に含め、画面は「最新＋数フレーム」から始める。

## レビュー反映（rev.3）

- フェイクの**誤本番有効化**を要件レベルで禁止し、**設定ガード**を成功基準に近い強さで記載した。
- **契約一貫性**（メタとタイル）を成功基準に明記し、`03` の公開モデルへ委譲した。
- アニメーションを「後追いのみ」から **MVP 内の条件付き**に変更し、フレーム数・間隔でスコープクリープを抑えた。

## デプロイ反映（rev.7）

- **GitHub Pages ＋ Cloudflare Workers** を規範とした（`09`）。

## 最適化反映（rev.8）

- タイル取得保証の対象を **メタデータの `coverage` と `zoom_range`** で明示し、範囲外タイルを成功基準の対象外として扱えるようにした。
